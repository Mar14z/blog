# 静墨博客使用指南

一个极简风格的个人博客系统，基于 Node.js + MongoDB 构建。

## 快速开始

### 1. 安装依赖
```bash
npm install
```

### 2. 配置环境变量
复制 `.env.example` 为 `.env` 并修改配置：
```bash
cp .env.example .env
```

### 3. 启动 MongoDB
确保本地 MongoDB 运行在默认端口 27017，或修改 `.env` 中的 `MONGODB_URI`。

### 4. 启动服务
```bash
npm start
```

访问 http://localhost:3000

## 管理后台

访问 http://localhost:3000/admin

默认账号密码：
- 用户名: `admin`
- 密码: `admin123`

## 主要功能

### 文章管理
- 创建、编辑、删除文章
- 支持 Markdown 格式
- 自动生成摘录
- 分类和标签管理

### Obsidian 同步
系统支持与 Obsidian 笔记库双向同步。

详见 [Obsidian 双向同步指南](./obsidian-bidirectional.md)

### 图片上传
支持图片上传功能，通过管理后台上传或 API：
```
POST /api/upload/image
```

## 部署

### Docker 部署（推荐）
```bash
docker-compose up -d
```

### 手动部署
```bash
# Linux/Mac
./scripts/deploy.sh

# Windows
scripts\deploy.bat
```

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/articles` | GET | 获取文章列表 |
| `/api/articles/:slug` | GET | 获取单篇文章 |
| `/api/articles` | POST | 创建文章 |
| `/api/articles/:slug` | PUT | 更新文章 |
| `/api/articles/:slug` | DELETE | 删除文章 |
| `/api/upload/image` | POST | 上传图片 |
| `/api/health` | GET | 健康检查 |

## 常见问题

### Q: MongoDB 连接失败
检查 MongoDB 是否运行，或确认 `.env` 中的 `MONGODB_URI` 配置正确。

### Q: 文章不显示
1. 确认文章 `published` 状态为 `true`
2. 检查分类是否正确
3. 查看管理后台的文章列表

### Q: 图片上传失败
检查 `uploads` 目录权限，确保服务器有写入权限。
